Discuss the scope of var, let, and const

  • var - Function Scoped (or Global):
  • If declared within a function, the variable is accessible only inside that function.
  • If declared outside any function, it's globally accessible.
  • var can be redeclared and reassigned, potentially leading to confusion.
  • let - Block Scoped:
  • Declared within any block of code delimited by curly braces ({}).
  • Accessible only within its immediate block or nested blocks.
  • Can be reassigned but not redeclared within the same block.
  • const - Block Scoped (Immutable):
  • Similar to let in terms of block scoping.
  • Must be assigned a value during declaration.
  • The value cannot be reassigned later. Objects declared with const can have their properties modified, but you can't reassign the entire object.
  • Best Practices:
  • Prefer let and const for most cases to avoid issues with var's overly broad scope and accidental redeclarations.
  • Use const for values you know should never change.

Use Cases of null and undefined

  • undefined:
  • Automatically assigned to declared variables that don't yet have a value.
  • Also assigned to missing object properties or function return values without explicit return statements.
  • Generally indicates the absence of a defined value.
  • null:
  • A developer must explicitly assign null to a variable.
  • Used to represent a deliberate "empty" or "no value" state.

What is a REST API?

  • REST (Representational State Transfer): An architectural style for designing web APIs.
  • API (Application Programming Interface): A set of rules that define how different pieces of software can interact with each other.
  • Key Principles of REST APIs:
  • Client-Server: Separation of concerns (frontend client, backend server).
  • Stateless: The server doesn't maintain client state between requests.
  • Uniform Interface: Resources are identified by URLs, and standard HTTP methods (GET, POST, PUT, DELETE) are used for manipulation.
  • Why REST APIs Matter:
  • Scalability, Interoperability: Widely used due to standardized communication patterns, making it easy for different systems to interact.
  • Example:
  • Getting user data with ID 123 from a REST API:
  • HTTP Request: GET /api/users/123
